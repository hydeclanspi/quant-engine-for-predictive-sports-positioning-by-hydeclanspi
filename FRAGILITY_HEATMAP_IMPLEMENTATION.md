## 🎨 UI 设计规格

### 矩阵热力图
```
           比赛1  比赛2  比赛3  比赛4
       ┌────────────────────────────┐
比赛1  │  -    24%   58%   42%      │
比赛2  │  24%   -    35%   45%      │
比赛3  │  58%  35%    -    61%      │
比赛4  │  42%  45%   61%    -       │
       └────────────────────────────┘
```

**特点**：
- 上三角矩阵（下三角为空白）
- 每个单元格 12×12px（响应式）
- 显示脆弱性分数和百分比
- 悬停显示工具提示：`比赛 i × 比赛 j: XX% 🚨`

### 颜色编码

| 范围 | 颜色 | 含义 | Tailwind |
|------|------|------|----------|
| 0-25% | 🟢 绿色 | 安全 | `bg-emerald-100` `border-emerald-300` |
| 25-40% | 🟡 黄色 | 注意 | `bg-amber-100` `border-amber-300` |
| 40-60% | 🟠 橙色 | 风险 | `bg-orange-100` `border-orange-300` |
| 60-100% | 🔴 红色 | 危险 | `bg-red-100` `border-red-300` |

### 风险指示符
```
✅ 绿色  → LOW (0-25%)
⚠️  黄色  → MEDIUM (25-40%)
⛔ 橙色  → HIGH (40-60%)
🚨 红色  → CRITICAL (60-100%)
```

## 🔄 数据流

```
用户选择比赛
    ↓
checkedMatches 更新
    ↓
FragilityHeatmapCard 接收 matches prop
    ↓
useMemo 计算：
  1. 从已结算投资提取历史数据
  2. 对每个比赛对调用 assessComboFragility()
  3. 生成脆弱性矩阵
    ↓
组件渲染：
  1. 矩阵网格（上三角）
  2. 颜色编码单元格
  3. 统计面板
    ↓
用户点击单元格
    ↓
setExpandedFragilityPair() 更新
    ↓
详细信息面板展开（带动画）
```

## 📊 数据处理

### 历史数据提取
```javascript
const historicalData = useMemo(() => {
  const investments = getInvestments()
  return investments
    .filter(inv => inv.is_settled === true)  // 仅已结算投资
    .slice(0, 100)                           // 限制前100个（性能）
    .map(inv => ({
      matches: inv.matches.map(m => ({
        odds: Number(m.odds) || 1,
        result: m.status === 'won'           // 比赛胜负
      })),
      succeeded: inv.status === 'won',       // 整个投注胜负
      createdAt: inv.createdAt              // 时间戳
    }))
}, [])
```

### 脆弱性计算
```javascript
const assessment = assessComboFragility(
  [matches[i], matches[j]],  // 2个比赛的组合
  historicalData             // 历史数据
)
// 输出：{
//   overallFragility: 42.37,
//   pairAnalysis: [{...}],
//   ...
// }
```

## 🎯 交互流程

### 1️⃣ 初始状态
- 用户在左侧选择比赛
- `checkedMatches` 更新
- 热力图自动计算和渲染

### 2️⃣ 查看矩阵
- 实时显示所有比赛对的脆弱性分数
- 颜色编码：直观看到风险分布

### 3️⃣ 点击详情
- 点击矩阵中的任何单元格

  - 风险等级

- 建议："考虑避免同时投注或分割成独立组合"
- 用户可基于此调整投注策略

## 🔧 技术实现细节

### 组件结构
```jsx
FragilityHeatmapCard
├── 标题区域
│   ├── 图标 + 名称
│   ├── 高风险对警示标签
│   └── 图例文本
├── 矩阵热力图网格
│   ├── 列标题（1, 2, 3, ...）
│   ├── 行标题（1, 2, 3, ...）
│   └── 单元格网格
│       ├── 下三角（空白）
│       └── 上三角（脆弱性分数）
├── 统计面板
│   ├── 颜色图例
│   └── 关键指标
│       ├── 平均脆弱性
│       ├── 高风险对数
│       └── 历史样本量
└── 详细信息面板（展开时）
    ├── 比赛对编号和赔率
    ├── 脆弱性分数
    ├── 风险等级
    ├── 置信度
    ├── 统计显著性
    └── 警告和建议
```

### 关键 Props

```javascript
<FragilityHeatmapCard
  matches={[                          // 用户选择的比赛
    { odds: 5.5, teamName, matchDate },
    { odds: 2.0, teamName, matchDate },
    ...
  ]}
  expandedPair={null}                 // 当前展开的比赛对
  onSelectPair={(pair) => {}}         // 点击时的回调
/>
```

### 性能优化

1. **useMemo 缓存**
   - 历史数据提取（仅在初始化时）
   - 脆弱性矩阵计算（matches 变化时重算）

2. **样本量限制**
   - 限制历史数据前 100 个投注
   - 保持快速计算（<50ms）

3. **条件渲染**
   - 少于 2 个比赛时显示占位符
   - 只计算必要的矩阵单元格
